# 🚀 AtCoder ABC 統合ツール（簡潔版）

## 📋 概要
AtCoder ABC用の**シンプルで強力な**統合ツールです。  
1つのコマンドですべてが完結します。

## 🎯 主な機能
- 📁 **フォルダ作成**: テンプレート付きコンテストフォルダ自動生成
- 🧪 **テスト実行**: 複数テストケース対応・TLE検出
- ⏰ **タイマー**: リアルタイム残り時間表示
- 🌐 **URL開く**: 全問題を瞬時にブラウザで開く

---

## 🚀 基本的な使い方

### コマンドライン（推奨）
```bash
abc new 372          # ABC372フォルダ作成
abc test a py        # A問題をPythonでテスト  
abc test b cpp       # B問題をC++でテスト
abc timer            # タイマー開始
abc open 372         # 全問題URL開く
```

### PowerShell直接実行
```powershell
.\abc.ps1 new 372    # 同様の機能
.\abc.ps1 test a py
.\abc.ps1 timer
```

```
├── main.cpp         # C++テンプレート

```

**たったこれだけ！** 複雑なファイルは一切なし。

## 🎮 実際のコンテストでの使用例

### 1. **コンテスト準備**
```bash
abc new 372          # フォルダ作成
cd ABC372           # フォルダに移動
abc open 372        # 全問題を開く（ABC直下から実行）
```

### 2. **フォルダ内での作業**
```bash
# ABC372フォルダ内で
run test a py       # A問題をPythonでテスト
run test b cpp      # B問題をC++でテスト  
run timer           # タイマー開始
run open            # 全問題URL開く
```

## 🔧 初期設定

1. **PowerShell実行ポリシー設定**（初回のみ）
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```

2. **C++環境**（使用する場合）
   - g++がパスに通っていること
   - AtCoder Libraryがインストール済みであること

## 🎯 設計思想

### ❌ 従来の問題点
- ファイルが多すぎて混乱
- コマンドが長くて覚えられない  
- 機能が分散していて使いにくい

### ✅ 新しいアプローチ
- **1つのコマンド**ですべて完結
- **最小限のファイル**で最大の機能
- **短いコマンド**で高速操作
- **フォルダ内でも外でも**同じように動作

## 🚨 トラブルシューティング

### Q: コンパイルエラーが出る
```
❌ コンパイルエラー: 'bits/stdc++.h' file not found
```
**A**: g++とAtCoder Libraryを確認してください

### Q: テストケースが認識されない  
**A**: `## Test 1` と `## Expected:` の形式を確認

### Q: Pythonが見つからない
**A**: `python`コマンドが利用可能か確認（`python --version`）
