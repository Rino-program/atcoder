AtCoder Template 仕様ガイド

template.py を本番で迷わず使うための実用ドキュメント

概要 最速セットアップ 入出力・定数 頻出パターン 組み合わせレシピ 関数リファレンス 注意点 提出前チェック

概要

  • このテンプレートは、AtCoderでよく使うアルゴリズムとデータ構造を1ファイルで即利用できるようにしたものです。
  • 入力ヘルパ、グラフ、木、Union-Find、セグ木、文字列処理、二分探索、座標圧縮までを網羅しています。
  • 多くの関数に日本語ドキュメントが付いているため、競技中の再確認にも向いています。

前提: 0-indexed、区間は [l, r)（半開区間）を多用します。

最速セットアップ（本番向け）

  1. 問題ごとの main?.py に template.py の内容をコピー。
  2. main() 内の「ここに解答を書く」を編集。
  3. 不要なクラス・関数を削る必要はありません（提出サイズ制限内なら可）。
  4. テスト: oj test -c "python main?.py" -d input/?
  5. 提出: oj submit ... main?.py
最小テンプレ利用例 
def main() -> None:
    N = INT()
    A = LIST()
    print(sum(A))

if __name__ == "__main__":
    main()

入出力・定数・基本ユーティリティ

入力ヘルパ

  • INT(): int 1個
  • MAP(): 行の整数列を map で取得
  • LIST(): 行の整数列を list で取得
  • LISTS(n): n行の整数リスト
  • LISTSI(n): n行1整数
  • STR(), STRS(n), CHARS(), STRSL(n)

定数と出力

  • INF = 10**18
  • MOD = 998244353（必要なら 1e9+7 に変更）
  • Yes/No 系の定型出力関数
  • Output クラスでバッファ出力
  • debug() は標準エラー出力

頻出パターン（実戦で使う形）

1) グラフ最短路

重みなし: bfs、重み0/1: zero_one_bfs、非負重み: dijkstra を使い分け。

2) 連結判定・統合

辺追加と連結管理は DSU。差分制約なら WeightedDSU。

3) 区間クエリ

一点更新＋区間取得は SegTree、区間更新が必要なら LazySegTree。

4) 数え上げ

組合せは Combination、転倒数は inversion_count で即計算。

典型: 座圧 + BIT で順序統計 
A = LIST()
C = compress_list(A)
bit = BIT(max(C) + 1)
ans = 0
for i, x in enumerate(C):
    ans += i - bit.sum(x)
    bit.add(x, 1)
print(ans)

組み合わせレシピ（前提→使用関数→出力）

「どの関数をどう繋ぐか」を先に決めると実装が早く安定します。各レシピはそのまま貼って使える最小例です。

関数リファレンス

各項目に「使う条件」「最小コード例」「組み合わせる相手」を載せています。検索で絞り込んでから例をコピーしてください。

注意点（バグを減らす）

  • 添字: ほぼ0-index。問題文が1-indexなら入力時に -1 する。
  • 区間: 多くが [l, r) の半開区間。端の off-by-one に注意。
  • 距離配列: BFS は未到達=-1、Dijkstra系は未到達=INF。
  • 前処理依存: 例として dijkstra は build_weighted_graph と組み合わせると実装ミスが減る。
  • 再帰: sys.setrecursionlimit(10**6) は設定済みだが、深すぎる再帰は iterative 化を検討。
  • MOD: 問題によって 998244353 / 1e9+7 を切り替える。
  • 重み付き辺: 負辺があると Dijkstra 不可。Bellman-Ford か別解法を使う。
