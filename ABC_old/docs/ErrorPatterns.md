# 🚨 競技プログラミング よくあるエラー & 対処法

## 📊 頻出エラーランキング

### 🥇 1位: オーバーフロー
**症状**: 答えが間違っている、負の数が出る
**原因**: `int` の範囲を超えた計算
**対処法**:
```cpp
// ❌ 悪い例
int a = 1e9, b = 1e9;
int result = a * b;  // オーバーフロー

// ✅ 良い例
long long a = 1e9, b = 1e9;
long long result = a * b;
```

### 🥈 2位: 配列の範囲外アクセス
**症状**: Runtime Error (RE)、不正な値
**原因**: 配列のサイズを超えたアクセス
**対処法**:
```cpp
// ❌ 悪い例
vector<int> a(n);
for(int i = 1; i <= n; i++) {  // i=n でアクセス違反
    a[i] = i;
}

// ✅ 良い例  
vector<int> a(n);
for(int i = 0; i < n; i++) {
    a[i] = i + 1;
}
```

### 🥉 3位: TLE (Time Limit Exceeded)
**症状**: 時間制限超過
**原因**: 計算量が大きすぎる
**対処法**:
```cpp
// ❌ 悪い例: O(N²) ソート
for(int i = 0; i < n; i++) {
    for(int j = i+1; j < n; j++) {
        if(a[i] > a[j]) swap(a[i], a[j]);
    }
}

// ✅ 良い例: O(N log N) ソート
sort(a.begin(), a.end());
```

## 🎯 言語別注意点

### C++
1. **整数除算**: `7/2 = 3` (小数点切り捨て)
2. **負数の剰余**: `(-5) % 3 = -2` (Pythonと異なる)
3. **文字列の比較**: 辞書順比較可能
4. **vectorの初期化**: `vector<int> a(n, 0)` で0埋め

### Python  
1. **再帰制限**: `sys.setrecursionlimit(10**7)` 必須
2. **PyPy3推奨**: CPythonは遅い場合がある
3. **整数除算**: `7//2 = 3`, `7/2 = 3.5`
4. **リスト初期化**: `[[0]*m for _ in range(n)]` (2次元)

## ⚡ パフォーマンス最適化

### 計算量の目安
| 制限時間 | 許容計算量 | 例 |
|----------|------------|-----|
| 1秒 | ~10⁸ | N≤10⁴のO(N²) |
| 2秒 | ~2×10⁸ | N≤10⁵のO(N log N) |
| 3秒 | ~3×10⁸ | N≤10⁶のO(N) |

### よくある最適化
1. **入出力高速化**: `ios_base::sync_with_stdio(false)`
2. **不要なコピー削除**: 参照渡し `const vector<int>&`
3. **定数倍最適化**: ループアンローリング、ビット演算

- 最大値: N=制限値, A[i]=最大値

## 📚 問題パターン別対策

### グラフ問題
- 連結性: DSU (Union-Find)
- 最短路: BFS (重みなし), Dijkstra (重みあり)
- トポソート: DAGの順序決定

### DP問題
- メモ化: `dp[i][j] = -1` で初期化、計算済みかチェック
- 状態設計: 「i番目まで見て、条件jを満たす場合の最適値」

### 数学問題
- MOD計算: 毎回 `% MOD` を忘れずに
- 組み合わせ: 事前計算で O(1) 計算
- 素数判定: エラトステネスで前処理
